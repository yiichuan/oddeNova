# 结构化音乐制作流程

> 面向 Strudel 实时编码环境，从专业音乐制作人视角设计的分层工作流。
> 每个阶段定义明确的输入、输出和验收标准，方便 Agent 逐步推进。

---

## 核心理念

专业制作人不会同时处理所有元素。他们遵循一个从**骨架到血肉**的建造顺序：

```
方向 → 节奏 → 低频 → 和声 → 旋律 → 氛围 → 混音
```

每一层都建立在上一层之上，且只做一件事。这个顺序不是随意的——它遵循音乐中频率和能量的优先级。

---

## 阶段 0：方向确定（Direction）

在写任何一个音符之前，先明确四个参数。

### 必须确定的参数

| 参数 | 说明 | 常见选项 |
|------|------|----------|
| **风格** | 音乐类型 | lo-fi / house / techno / trap / ambient / drum & bass / jazz |
| **情绪** | 情感基调 | dark / uplifting / melancholic / aggressive / dreamy / tense |
| **速度** | BPM | 60–80（lo-fi）/ 120–128（house）/ 138–145（techno）/ 140–160（D&B）|
| **调性** | 根音 + 调式 | C minor / A major / F dorian / D phrygian… |

### Strudel 映射

```javascript
// 速度：setcps(BPM / 120 * 0.5)
// 120 BPM → setcps(0.5)
// 140 BPM → setcps(0.583)
// 90 BPM  → setcps(0.375)

// 调性在旋律/和声阶段通过 .scale("C4:minor") 等约束
```

### Agent 任务

> 从用户描述中提取这四个参数。若用户只说「一段深邃的 techno」，则推断：  
> 风格=techno, 情绪=dark, BPM=140, 调性=A minor（techno 惯用）  
> 在 reasoning 中向用户确认这些参数。

## 阶段 1：节奏骨架（Rhythm Foundation）

节奏是一切的地基。先建鼓，才能听出其他层是否合适。

### 四个鼓组声部

```
主拍（Kick）   ──────── 定义律动感和 BPM 的视觉重心
军鼓/拍手（Snare/Clap） ─ 定义律动的反拍张力
踩镲（Hi-hat） ──────── 填充细节，控制能量密度
打击乐（Perc） ──────── 可选，增加groove感和切分
```

### 典型模式示例

```javascript
// House 四四拍
setcps(0.5)
stack(
  s("bd*4"),                           // kick：每拍一下
  s("~ cp ~ cp"),                      // clap：2、4拍
  s("hh*8").gain(0.5),                 // hi-hat：八分音符
)

// Trap 模板
setcps(0.583)
stack(
  s("bd ~ ~ bd ~ bd ~ ~"),             // kick：切分
  s("~ ~ sd ~ ~ ~ sd ~"),              // snare
  s("hh*16").gain(0.4),                // hi-hat：十六分音符
  s("808oh(3,8)").gain(0.6),           // open hat：欧几里得节奏
)

// Lo-fi 慵懒感（带摇摆）
setcps(0.375)
stack(
  s("bd ~ bd ~").bank("RolandTR808"),
  s("~ sd ~ sd").bank("RolandTR808"),
  s("hh(5,8)").gain(0.45),            // 不规则 hi-hat
)
```

### 验收标准

- [ ] 单独播放鼓组，律动感清晰，能感受到 BPM
- [ ] 闭眼听，能判断是什么风格的鼓机
- [ ] 没有元素在音量上互相掩盖

### Agent 任务

> 只写鼓组，忽略其他一切。通过 evaluate_code 测试。
> 失败时重点检查：`setcps` 是否单独成行，引号是否配对。

## 阶段 2：低频层（Low End）

低频层是鼓组之上第一个加入的音高元素。它的音符决定了整首曲子的和声走向。

### 低频的三种角色

```
Sub Bass  ── 纯粹的低频能量，通常是根音，单音为主（sine / 808bd）
Mid Bass  ── 有音色的贝斯线，可以有旋律性（sawtooth / bass sample）
808       ── 有音高的鼓机贝斯，介于 Sub 和 Mid 之间（808bd + note）
```

### 低频与鼓组的配合规则

1. **Bass 和 Kick 不能同时争夺同一频段**：kick 打的时候 bass 可以短促一些，或用 side-chain 压缩感（在 Strudel 中用 `.gain()` 的节奏模拟）
2. **Bass 的根音 = 和弦进行的根音**
3. **低频越简单越好**：8 个音符的 bass line 通常比 32 个音符的更有力量

### 典型模式示例

```javascript
// 简单根音贝斯（house 风格）
note("c2 c2 c2 c2 eb2 eb2 f2 f2")
  .s("sawtooth")
  .lpf(400)
  .gain(0.7)
  .attack(0.01)
  .release(0.1)

// 律动贝斯（funk / trap）
note("c2 ~ eb2 ~ f2 ~ ~ eb2")
  .s("bass")
  .gain(0.8)

// 808 滑音贝斯
note("c1 ~ ~ eb1").s("808bd").gain(0.9).decay(0.8)
```

### 验收标准

- [ ] 和鼓组一起听，低频不会糊在一起
- [ ] bass line 的根音走向能被感知（即使模糊也行）
- [ ] 低频层和鼓组加起来已经有完整的「groove」感

### Agent 任务

> 根据阶段 0 确定的调性，选择 2–4 个根音写 bass line。
> 新 pattern 加入现有 `stack()` 中，不改动鼓组。
> 测试时若报错，优先检查 `note()` 音高写法。

## 阶段 3：和声层（Harmony）

和声层赋予音乐情感色彩。它在频率上介于低频和旋律之间，通常是和弦。

### 和弦的四种形态

```
长音 Pad      ── 缓慢变化，像底色（.sustain(2).room(3)）
切分 Stab     ── 短促有力，打在节拍缝隙（.decay(0.1).attack(0.01)）
琶音 Arp      ── 和弦音依次演奏（用 scale + n() 实现）
采样 Chop     ── 采样切片重组（.speed() + pattern 控制）
```

### 常用和弦进行

```javascript
// 小调 i–VI–III–VII（流行/电子通用）
note("<[c3,eb3,g3] [ab2,c3,eb3] [eb3,g3,bb3] [bb2,d3,f3]>")
  .s("piano").gain(0.5).room(2)

// Dorian i–IV（house / funk）
n("<0 3>").scale("C4:dorian")
  .s("juno").gain(0.45).sustain(0.8)

// 小调 i–iv–V（古典张力）
note("<[c3,eb3,g3] [f3,ab3,c4] [g3,b3,d4]>")
  .s("arpy").gain(0.4).delay(0.3).room(2)
```

### 验收标准

- [ ] 和弦进行在 2–4 个小节内循环，有内在逻辑
- [ ] 和鼓 + 低频一起听，三层不打架
- [ ] 闭眼听，能感受到情绪（dark / hopeful / tense…）

### Agent 任务

> 根据阶段 0 的调性和情绪，选择合适的和弦进行。
> 和声层音量（.gain）默认比 bass 低 10–15%。
> 注意：和弦 note() 内用逗号表示同时发音，如 `note("c3,eb3,g3")`。

## 阶段 4：旋律层（Melody）

旋律是听众记住的东西。一条好的 lead melody 应该有「呼吸」——有音符，也有空白。

### 旋律的三个层次

```
Lead（主旋律）   ── 最突出，频率在 500Hz–4kHz，听众跟着哼的部分
Counter（对位）  ── 在 Lead 的空隙里回应，增加对话感
Hook（钩子）    ── 最短最洗脑的动机，通常只有 2–4 个音符
```

### 旋律写作原则

1. **留白比音符更重要**：`~` 是你最好的朋友
2. **重复是记忆的基础**：4 小节的动机至少重复 2 遍才能被记住
3. **方向感**：旋律线需要有起伏，不能一直在同一音区
4. **音阶约束**：用 `.scale()` 保证不走调

```javascript
// 简单 hook（容易被记住）
n("0 ~ 4 ~ 7 ~ 4 ~").scale("C4:minor")
  .s("arpy").gain(0.6).decay(0.3)

// 有呼吸的旋律线
n("0 2 4 ~ 7 4 2 ~").scale("A4:minor pentatonic")
  .s("pluck").gain(0.55).room(1.5)

// 对位旋律（在主旋律休息处填充）
n("~ ~ 2 4 ~ ~ 5 ~").scale("C4:dorian")
  .s("sine").gain(0.35).delay(0.4)
```

### 验收标准

- [ ] 单独循环播放旋律，8 小节内能哼出来
- [ ] 和全层合并后，旋律仍然清晰（不被淹没）
- [ ] 有明显的「起」「承」「转」「合」感

### Agent 任务

> 优先写简单的 hook（2–4 音符），而非复杂旋律线。
> 旋律必须和阶段 3 使用相同的调性（`scale` 一致）。
> 旋律的 `.gain()` 应高于和声层，低于或等于 kick。

## 阶段 5：氛围与纹理（Atmosphere）

这一层是「空气」。它让音乐有深度和空间感，是专业和业余制作人最大的差距之一。

### 三种氛围元素

```
Pad（铺垫）    ── 长音、慢攻击，作为背景底色，大量混响
Texture（纹理）── 不规则的声音碎片，增加真实感和意外性
Riser/Sweep  ── 能量堆积或泄放，用于过渡（Strudel 中可用 .slow()）
```

```javascript
// 大 Pad（放在最底层）
note("<c2 ab1>").s("pad")
  .gain(0.3).room(4).delay(0.6).sustain(4).attack(1)

// 噪声纹理
s("wind").gain(0.15).room(3).slow(2)

// 金属碎片纹理
s("metal(3,16)").gain(0.2).room(2).pan("0.2 0.8 0.5")

// hi-hat 泛音氛围
s("hh*16").gain(0.08).hpf(8000).room(4)
```

### 验收标准

- [ ] 单独静音氛围层，整体感觉「干」「空」
- [ ] 加入氛围层后，感觉在一个空间里（房间/教堂/户外）
- [ ] 氛围层不抢主角（永远是配角）

### Agent 任务

> 氛围层 `.gain()` 通常在 0.1–0.3 之间。
> 必须有 `.room()` 或 `.delay()`，否则不算真正的氛围层。
> 如果已有 5 层以上，此阶段可以跳过。

---

## 阶段 6：混音平衡（Mix Balance）

混音不是加效果，是让每个层都有自己的「位置」而不互相干扰。

### 三维空间模型

```
纵轴（音量层级）     横轴（立体声位置）     深轴（远近/混响）
─────────────────   ─────────────────     ─────────────────
Kick    最响         Kick    中心          Kick    最近（干）
Bass    其次         Bass    中心          Bass    较近
Snare   响           Snare   轻微宽        Snare   略有房间感
Lead    突出         Lead    中心偏左/右   Lead    中近
Pads    安静         Pads    宽（.jux）    Pads    最远（多混响）
```

### Strudel 混音工具

```javascript
// 音量层级参考（相对值）
kick.gain(0.9)
bass.gain(0.75)
snare.gain(0.7)
lead.gain(0.65)
chord.gain(0.45)
pad.gain(0.25)
texture.gain(0.15)

// 立体声
.pan(0.3)           // 偏左
.pan(0.7)           // 偏右
.jux(rev)           // 左右分别播放正向/反向版本

// 频率分离（让各层占据不同频段）
bass.lpf(400)       // 只保留低频
snare.hpf(200)      // 去除不必要的低频
pad.lpf(2000)       // 去除刺耳高频
lead.hpf(500)       // 让旋律更清晰
```

### 验收标准

- [ ] 任意两层叠加，都不会有音量上的「冲突」感
- [ ] 全层播放，低频清晰，高频不刺耳，中频旋律突出
- [ ] 立体声有宽度，不是所有东西都挤在中间

### Agent 任务

> 检查现有所有层的 `.gain()` 值是否符合上方层级参考。
> 检查有无多个层缺少频率分离（.lpf / .hpf）。
> 调整时只改数值，不改音符和节奏。

## 完整代码结构模板

按阶段完成后，代码应具有以下结构（含注释标注每层角色）：

```javascript
// ── 方向：house / uplifting / 124 BPM / C minor ──
setcps(0.517)

stack(
  // ── 阶段1：节奏骨架 ──
  s("bd*4").gain(0.9),
  s("~ cp ~ cp").gain(0.7),
  s("hh*8").gain(0.45).hpf(6000),

  // ── 阶段2：低频 ──
  note("c2 c2 eb2 f2").s("sawtooth").lpf(380).gain(0.72).attack(0.01).release(0.08),

  // ── 阶段3：和声 ──
  note("<[c3,eb3,g3] [ab2,c3,eb3] [eb3,g3,bb3] [bb2,d3,f3]>")
    .s("juno").gain(0.42).room(1.5).sustain(1),

  // ── 阶段4：旋律 ──
  n("0 ~ 4 ~ 7 4 ~ 2").scale("C4:minor")
    .s("arpy").gain(0.6).decay(0.25).delay(0.2),

  // ── 阶段5：氛围 ──
  note("<c2 ab1>").s("pad").gain(0.22).room(4).attack(1.5).sustain(4),
)
```

## Agent 实施策略

### 渐进式建造原则

Agent 每次只做一个阶段，按顺序推进。禁止跳阶。

```
用户：「来一段 lo-fi」
Agent：阶段0确认参数 → 阶段1写鼓 → 测试 → 阶段2写bass → 测试 → ...
```

### 层识别方法

Agent 在收到当前代码时，应先分析已有哪些层：

```
已有 bd/sd/hh/cp 相关 s()    → 节奏骨架已完成
已有低音区 note()（c1/c2/d2）→ 低频层已完成
已有和弦（note 含逗号）      → 和声层已完成
已有高音区单音 note()        → 旋律层已完成
已有 .room(>2) 的长音 pad   → 氛围层已完成
```

### 单次对话任务拆分

| 用户描述 | Agent 行动 |
|----------|-----------|
| 「来一段 X 风格」（从零开始）| 阶段 0→1，仅写鼓，说明下一步 |
| 「加贝斯」 | 识别当前层 → 执行阶段 2 |
| 「加一点氛围感」 | 识别当前层 → 执行阶段 5 |
| 「换 house 风格」 | 重建阶段 1（鼓改变），保留音高层结构 |
| 「让它更暗」 | 修改阶段 3（和弦改小调/Phrygian）+ 阶段 5（pad 增强）|
| 「整体混音太糟了」 | 仅执行阶段 6，不改音符 |

### 每层测试 checklist

Agent 在每次 `evaluate_code` 后，除了检查语法，还应验证：

```
阶段1：鼓的 gain 在 0.5–0.9 之间？setcps 是否单独一行？
阶段2：bass 的 note 是否在 1–2 八度范围（c1–c3）？有 lpf 吗？
阶段3：和弦是否与 bass 根音一致？gain < 0.55 吗？
阶段4：旋律 scale 与阶段3一致吗？有没有足够的 ~ 留白？
阶段5：pad 有 .room() 吗？gain < 0.35 吗？
阶段6：各层 gain 是否符合层级关系？
```

## 风格速查表

| 风格 | BPM | 调式 | 鼓特征 | 贝斯特征 | 氛围特征 |
|------|-----|------|--------|----------|----------|
| Lo-fi Hip-Hop | 70–90 | minor / dorian | 慵懒，swing hi-hat | 短促，带 vinyl 感 | 大量 room，模糊 |
| Deep House | 120–124 | minor / dorian | 四四拍 kick，open hat | 律动性强，跟 kick | warm pad，长混响 |
| Techno | 138–145 | minor / phrygian | 硬 kick，工业感 | 极简，sub bass | 金属噪声纹理 |
| Trap | 130–160（半速=65–80）| minor | 滚动 hi-hat，深 808 | 808 滑音 | 空旷 reverb |
| Drum & Bass | 170–180 | minor | Amen break 风格，切分重 | 跳跃，中频 | 黑暗 pad |
| Ambient | 60–90 | major / lydian | 几乎无鼓或极稀疏 | 几乎无 | 全部都是氛围 |
| Jazz-inflected | 90–110 | dorian / lydian | swing，刷片 | walking bass | 温暖 delay |
