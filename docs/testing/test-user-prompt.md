## 测试维度映射

### 覆盖范围

| 维度 | 包含的用例 | 说明 |
|-----|----------|------|
| 完整作品生成 | TC-001, TC-002, TC-003 | 一次性生成完整可播放的作品 |
| 风格理解与生成 | TC-004 ~ TC-008 | 系统对不同音乐风格的理解程度 |
| 场景映射能力 | TC-009 ~ TC-012 | 从具体场景或参考到音乐美学的转换 |
| 参数精确控制 | TC-013 ~ TC-015 | 用户对 BPM、和弦、乐器等技术参数的精确指定 |
| 开放式理解 | TC-016 ~ TC-018 | 系统对极简输入的智能补全和默认化处理 |

### 用户提示词分类

| 分类 | 特点 | 对系统的考验 |
|-----|------|-----------|
| 自然表达版 | 日常语言 + 详细技术指标混合 | 需要同时理解自然语言和参数规范 |
| 极简触发版 | 依赖上下文和风格参考 | 需要强 few-shot 对齐能力 |
| 定制变体版 | 基于已知范本的参数替换 | 需要理解结构并灵活替换 |
| 风格/情绪类 | 主要传达情感和氛围 | 需要风格转换和情绪识别 |
| 参考场景类 | 从具体场景推导音乐特征 | 需要场景理解和映射能力 |
| 技术细节类 | 高度参数化的指定 | 需要音乐制作知识和参数精度 |
| 极简挑战类 | 最小信息量输入 | 需要强大的默认化和智能补全 |

## 验收标准

系统需要在 one-shot 模式下产出 **coastline 级别** 的代码，具体指标：

- [ ] TC-001 ~ TC-003：生成的作品应包含所有指定的结构元素（和弦、乐器层、效果）
- [ ] TC-004 ~ TC-012：生成的音乐能清晰传达风格/情绪/场景特征
- [ ] TC-013 ~ TC-015：参数（BPM、和弦、乐器）应精确应用
- [ ] TC-016 ~ TC-018：极简输入应生成听起来自然且完整的音乐
